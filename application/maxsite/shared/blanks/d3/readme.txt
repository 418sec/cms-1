Методика создания шаблона без style-all-mini.css
------------------------------------------------
См. http://max-3000.com/page/d3 (Концепция шаблона D3)

1. Загрузка только шаблонных стилей решается путем создания css/style.php в своем шаблоне. 
	Этот файл автоматически создаёт style.css из исходного css-less/style.less

2. Исходные less-файлы размещаются в каталоге шаблона css-less. Основной файл style.less. Именно 
	в нём необходимо выполнять все подключения.

3. В css-less/30-maxsite/ размещены less-файлы, которые содержат стили, типовые для MaxSite CMS. 
	
	Важные:
	* mso-menu-vertical.less — основное меню сайта. 
	* mso-forms.less — класс .fform, используемый для построения форм.
	* mso-columns.less — css-классы для построения колонок.
	* mso-css-helpers.less — CSS-хелперы могут использоваться при создании записей. 
	
	Второстепенные:
	* mso-message.less — класс .message — сообщения/блоки. Используется в типографике.
	* mso-label.less — класс .label — метки, также используется в типографике.
	
	Каркас:
	* mso-style.less — содержит пустые каркасы по типовым блокам MaxSite CMS.
	
	Желательно включать все эти файлы в шаблон, поскольку они ориентируются на предопределенные 
	css-классы MaxSite CMS.

4. В css-less/40-plugins/ размещены less-файлы плагинов MaxSite CMS. Они содержат типовые 
	варианты стилей.

5. Схема подключения файлов в style.less определяется вебмастером. Предлагаемый вариант основан на 
	типовых рабочих этапах вёрстки сайтов.
	
	Для удобства, каталоги начинаются с цифрового обозначения уровня. Предполагается, что верстка 
	будет происходить поэтапно — от нижнего уровня до верхнего. Каждый уровень может содержать 
	под-уровни. Если уровень не используется, то не размещайте в нем файлов. Если какой-то less-файл 
	не требует использования, то переименуйте его, чтобы имя начиналось с «-» или «_». 
	
	10 — Уровень параметизированных (!) миксов. Эти миксы нужно явно вызвать на уровнях ниже.

	20 — Уровень «ядра». Здесь можно разместить css-фреймворк, сбросы браузера, переменные и т.д. 
		
		21 — Уровень CSS-хелперов.
		22 — Уровень типографики. 
		23 — Уровень модульной сетки.

	30 — Уровень CMS. MaxSite CMS содержит предопределенные css-классы. Их стили могут быть 
		обязательными (рекомендованными) или второстепенными.

	40 — Уровень плагинов. Стили каждого плагина следует располагать в отдельном файле.

	50 — Уровень меню. Меню выделено отдельно, чтобы была возможность создавать заготовки 
		и коллекции под разные дизайны. Кроме того, здесь могут быть меню с разной 
		функциональностью, например dropdown-меню.

	60 — Уровень компонентов шаблона. Каждый компонент в отдельном файле.  

	70 — Здесь размещаются все остальные блоки.
	

====================================================================================================
ПОРЯДОК СОЗДАНИЯ ШАБЛОНА (как вариант)
====================================================================================================

1. Создайте копию шаблона Default в каталоге будущего шаблона.
2. Удалите css/var_style.css.
3. Содержимое каталога css-less можно удалить, если он не будет использоваться.
4. Загрузить из shared/blanks/d3/ предлагаемые каталоги и файлы.

Организация less-файлы целиком ложится на вебмастера и может быть произвольной. 
Файл css-less/style.less считается основным, поэтому все подключения следует выполнять в нём.
